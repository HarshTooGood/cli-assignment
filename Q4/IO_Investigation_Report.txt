2. File Descriptor Observation

Command:

$ echo "Hello Linux" > sample.txt
$ ls -l /proc/$$/fd

Output:

total 0
lrwx------ 1 codespace codespace 64 Jun 30 18:36 0 -> /dev/pts/1
lrwx------ 1 codespace codespace 64 Jun 30 18:36 1 -> /dev/pts/1
lrwx------ 1 codespace codespace 64 Jun 30 18:36 2 -> /dev/pts/1
lrwx------ 1 codespace codespace 64 Jun 30 18:36 255 -> /dev/pts/1
l-wx------ 1 codespace codespace 64 Jun 30 18:36 6 -> /dev/null

Observation:

File descriptor 0 represents standard input (stdin), 1 represents standard output (stdout), 
and 2 represents standard error (stderr). Linux assigns these descriptors automatically to 
every process for handling input and output operations.

3. Output Redirection

Command:

$ echo "This is standard output" > output.txt
$ cat output.txt

Output:

This is standard output

Observation:

The > operator redirects standard output to a file. If the file already exists, its previous contents are overwritten.

4. Error Redirection

Command:

$ ls nonexistentfile 2> error.txt
$ cat error.txt

Output:

ls: cannot access 'nonexistentfile': No such file or directory

Observation:

The 2> operator redirects standard error to a file while preventing the error message from appearing on the terminal.

5. Output and Error Redirection

Command:

$ (ls sample.txt; ls nonexistentfile) > combined.txt 2>&1
$ cat combined.txt

Output:

sample.txt
ls: cannot access 'nonexistentfile': No such file or directory

Observation:

The 2>&1 syntax redirects standard error to the same destination as standard output, 
allowing both outputs to be stored in a single file.

7. Linux File I/O Management

Linux manages file I/O using file descriptors, which are non-negative integers assigned
to opened files and other input/output resources. 
Every process is provided with standard input (0), standard output (1), and standard error (2). 
The kernel handles all read and write operations through these descriptors, allowing input, output, 
and error streams to be redirected to files, devices, or other processes.

Conclusion

The investigation demonstrated how Linux manages file access and I/O operations using file descriptors. 
Open files were examined using lsof, file descriptors were observed through the /proc filesystem, 
output and error redirection were successfully tested, and process resource limits were inspected using ulimit. 
These experiments illustrate how the Linux kernel efficiently manages file input/output and process resources.
